# Design Bench — showcase web app

A small React + Vite + TypeScript site that presents the benchmark runs as one scrollable
page: each benchmark shows its grid image, with the full prompt and a per-model table
(status, time, output tokens, error reason, link to the actual generated page) tucked
behind expandable sections.

## Develop

```bash
# from the repo root, regenerate the data the app reads (after running benchmarks):
node scripts/build-web-data.mjs

cd web
npm install
npm run dev      # http://localhost:5173
npm run build    # → web/dist
```

The app renders entirely from `src/data/benchmarks.json` and assets in `public/` (both
generated by `scripts/build-web-data.mjs` and committed), so it builds without needing the
benchmark runner or any API keys.

## Deploy (Vercel)

The repo root has a `vercel.json` that builds this app:

```json
{ "buildCommand": "cd web && npm install && npm run build", "outputDirectory": "web/dist" }
```

Import the repo into Vercel (root = repo root) and it deploys as a static site. To refresh
the content, run a benchmark, run `node scripts/build-web-data.mjs`, commit, and push.
